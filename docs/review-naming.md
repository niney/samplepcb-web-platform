# 후기 명칭 정리 — 게시판 "사용후기" → "고객후기"

**결정일 2026-07-10.** 그누보드 게시판 `review`의 제목을 "사용후기"에서 **"고객후기"**로 변경했다. 코드 변경이 아니라 게시판 데이터(`g5_board.bo_subject`) 한 건 수정이며, 이 문서는 그 결정과 근거를 남긴다.

## 배경 — "사용후기"가 두 곳에 겹침

"사용후기"라는 명칭이 성격이 다른 두 기능에 동시에 붙어 혼란을 유발했다. 둘 다 내용상 정당한 "후기"라, 한쪽만 놔두고 다른 쪽을 바꾸는 식으로는 해소되지 않았다.

| 대상 | 원래 이름 | 성격 | 건수 |
|---|---|---|---|
| 그누보드 게시판 `review` | 사용후기 | 별점 없는 자유 서술 후기·감사글("잘 받았습니다"류), 관리자 Re: 답글 | 원글 43 (+답변 44) |
| 영카트 상품 별점후기 `g5_shop_item_use` → 이관본 `sp_review` | (상품)사용후기 | 별점 1~5, 주문/프로젝트 귀속, 구매확인 기반 | 61 (이관 완료) |

## 결정

- **게시판 `review`: `bo_subject` "사용후기" → "고객후기".** *누가* 쓰는가(고객)로 구분해, 자유 감상·감사 성격에 맞춘다. (다른 게시판 제목 관례에 맞춰 공백 없이 "고객후기".)
- **상품 별점후기**(`sp_review`): 코드 내부 통칭은 이미 **"별점후기"**로 확립(스키마 주석·커밋). 고객 노출 라벨은 추후 UI 구현 시 **"구매 후기"** 방향으로 잡되 이번 범위 아님(미확정).
- **"자유게시판"은 채택하지 않음** — 이미 `free` 게시판(`bo_subject`="자유게시판", 현재 0건)이 존재해 이름이 중복되고, 게시판 내용(후기·감사)과도 맞지 않는다. 범용 게시판이 필요하면 비어 있는 `free`를 활성화한다.

## 적용 범위·방법 (코어 무변경)

- `bo_subject`는 `g5_board` **DB 데이터**라 git으로 추적되지 않는다. 관리자 `/adm` 게시판설정에서 제목만 수정하면 된다(로컬 반영 완료).
- 테마에 `bo_table=review` **하드코딩 링크가 없다**(`grep` 0건). 게시판 제목·커뮤니티 최신글·메뉴가 `bo_subject`를 자동으로 따라가므로 **코드 수정이 불필요**하다.
- 그누보드 코어(`samplepcb-web/` subtree)는 손대지 않는다 — AGENTS.md §코어 비수정 원칙 준수.

## 운영 반영 (수동 필요)

로컬 `samplepcb` DB는 반영됐다. **운영 서버 DB에도 관리자 게시판설정에서 `review`의 제목을 "고객후기"로 동일하게 바꿔야 한다** — 데이터 변경이라 코드 배포로는 넘어가지 않는다.

## 상품 별점후기 노출 — `/reviews` (2026-07-10 구현)

영카트 상품 별점후기(`sp_review` 61건)를 공개 페이지 **`/reviews`** 에 노출한다. 표준 `/shop/itemuselist.php`는 ① `g5_shop_item_use` 0건(데이터가 `sp_review`로 변환 이관됨) ② 후기 `it_id`(레거시 50종) ∩ 타깃 상품(템플릿 5종)=0 INNER JOIN 으로 구조적 표시 불가라, `sp_review`를 직접 조회하는 신규 페이지 **`spcb/pages/reviews.php`** 를 만들었다(메인 슬라이드 `inc/main_slider.php`와 동형 브릿지). 코어·`.htaccess` 무변경 — `.htaccess` 규칙 2가 `/reviews` → `spcb/pages/reviews.php` 를 자동 리라이트한다.

- **안전**: 본문은 태그 전부 제거 → `html_entity_decode`(레거시 `&nbsp;` 등 엔티티 정규화) → 재이스케이프 → `nl2br` 로 XSS 차단. 작성자 실명 마스킹(박종현→박*현), **관리자 답변 본문도** 작성자 실명을 동일 규칙으로 마스킹. `mbId`(이메일 PII)는 미노출.
- **표시**: 별점·평균(★5.0)·총건수·최신순·페이지네이션(10/페이지). 노출 게이트 = `isConfirm=1`.
- **진입점**: `theme/sp-lite/inc/quicklinks.php` 의 '고객후기' → `/reviews`(임시 플로팅 패널). 정식 GNB 배치는 후속.
- **범위**: 읽기 전용 1단계. **관리자 노출토글(isConfirm)·고객 신규 작성은 후속.**
- **한계**: 답변 실명 마스킹은 작성자 본인 이름만 커버(답변이 제3자 이름을 부르면 미커버) — 신규 작성/관리 도입 시 정책 재점검.

## 홈 메인 게시판 정리 + 별점후기 쇼케이스 (2026-07-10 구현)

라이브 `www.samplepcb.co.kr`(리뉴얼된 새 디자인)의 메인 하단 구조를 sp-php 커뮤니티 홈(`theme/sp-lite/index.php`)으로 이관했다. 기존 홈은 그누보드 스톡 커뮤니티 레이아웃(`free/qa/notice` + `gallery` + "나머지 전 게시판 루프")이라 정리되지 않았다.

**메인 구조 = 슬라이더(기존) → 별점후기 쇼케이스 → 공지사항 | 질문답변 | FAQ 3단 그리드.**

- **별점후기 쇼케이스** — `theme/sp-lite/inc/main_reviews.php` 신규. `sp_review`(isConfirm=1) 최신 8건을 카드 그리드(4열/2열/1열 반응형)로. 평균 별점·총건수 헤더 + "전체 별점후기 보기" → `/reviews`. 라이브 "별점평가가 증명하는 제조서비스" 섹션 대응. **"사용후기" = 별점후기 쇼케이스(`sp_review`)** 로 확정(레거시 `review` 게시판 아님) — 라이브가 이 축을 씀. 이로써 `/reviews` 가 정식 진입점(MORE)을 얻음.
- **3단 게시판 그리드** — `notice`(공지사항)·`qa`(질문답변)·`faq`(FAQ). 신규 홈 전용 최신글 스킨 `theme/sp-lite/skin/latest/home/latest.skin.php` + `index.php` 인라인 CSS(색상 녹/청/회). 코어 `latest()` 재사용.
- **공용 헬퍼 추출** — `sp_review_mask/name/body/stars` + 신규 `sp_review_text`(평문 미리보기)를 `theme/sp-lite/inc/reviews_lib.php` 로 분리해 `reviews.php`·`main_reviews.php` 가 공유(중복 제거).
- **미배치 링크(작업용) 패널** — `inc/quicklinks.php` 에 '게시판' 그룹 추가: 홈에 안 올라간 나머지 게시판(`review` 고객후기·`data` 자료실·`customer_center` 고객센터·`open_market` 오픈마켓 + 0건인 `free`·`gallery`·`portfolio`·`production_s`). 관리자 전용 게이트 없이 전체 노출. 내장 FAQ(`faq.php`, `g5_faq` 항목 0건)는 홈 FAQ 게시판과 중복이라 패널에서 제거.
- **코어 무변경** — 신규는 전부 `theme/sp-lite/`(우리 테마)·`spcb/pages/`. `bbs/`·`shop/`·`config.php` 무수정. CSS 는 인라인 `<style>`(링크 CSS 아님)이라 `G5_CSS_VER` 캐시버스팅 불필요.
- **후속**: 별점후기 관리자 노출토글·고객 신규 작성(위 1단계 후속과 동일). `notice/qa/faq` 정식 GNB 메뉴 배치는 메뉴설정(DB) 영역.
